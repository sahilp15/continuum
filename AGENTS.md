Changes to these packages require review from another teammate before merge:

```text
packages/contracts
packages/config
```

## Collaboration rules

1. `main` is protected; all work lands via small PRs with acceptance criteria.
2. One issue per meaningful unit of work (see `docs/BACKLOG.md`).
3. Two coding agents never edit the same package simultaneously.
4. No merging failing tests; no bypassing type errors; no giant autogenerated commits.
5. Security-sensitive work requires review by a second person.
6. A complete vertical workflow is demonstrated at least twice per week.
7. Documentation is updated with any architecture change (ADRs for consequential decisions).
8. Incomplete integrations live behind feature flags; mocked functionality is always
   labeled mock — never "connected" or "live".
9. A cross-Space leakage test failure blocks release. Always.
